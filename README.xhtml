Introduction

This is an extremely rough source package. Geordi was primarily written for one specific use (namely as freenode/##c++ bot), and as such lacks things like proper configuration-, build-, and installation infrastructure. The installation procedure below is mostly just a reminder to myself.

Prerequisites

  • GNU/Linux on an i386 or x86_64 machine
  • GCC/G++ ≥ 4
  • Boost ≥ 1.33 (not strictly needed for the operation of geordi itself, but many prelude parts use Boost.)
  • GHC ≥ 6.6
  • utf8-string ≥ 0.1

    For those of us not familiar with GHC packages, to install this do:

    tar xf utf8-string-0.1.tar.gz
    cd utf8-string-0.1
    runhaskell Setup.lhs configure
    runhaskell Setup.lhs build
    su -c "runhaskell Setup.lhs install"

Optional step for improved error messages:

Do grep get_max_length /usr/include/c++/4.3.0/debug/formatter.h. If you don't get output, it means that your gcc version does not yet have configurable debug mode error formatter line width, which means that you should do the following manually: in that file, find the value 78, and change it into 780.

(See GCC bug 31518 for more information.)

Installation procedure:

The following is a list of steps needed to set up the environment geordi runs in. Don't copy/paste this into your terminal, this is not a shell script! It is intended to be read and interpreted by a human.

Assuming you extracted this package in some directory, cd into it and do:

  1. scripts/compile # To produce the ./Bot executable.
  2. Edit `config' to your liking.
  3. mkdir rt # This will be our chroot root.
  4. scripts/compile-prelude # To produce rt/prelude.h.gch and a few rt/foo.o's.
  5. rtcp () { cp -L -r -v --parents $@ rt ; }
  6. libs () { echo `ldd $@ | grep -o -P "(?<=\\s)/\\S*" | sort | uniq` ; }
  7. EXECS="`which g++ as ld strace` /usr/lib/gcc/i586-suse-linux/4.1.2/cc1plus" # Correct the latst one for your system.
  8. rtcp $EXECS `libs $EXECS`
  9. scripts/perms # To set up proper permissions for rt and selected files in it.
  10. sudo scripts/trycompile # This will keep giving errors about missing .o files and libraries. Copy those into rt using rtcp until rt/a.out is produced successfully.
  11. rtcp `libs rt/a.out`
  12. sudo chroot rt /a.out && echo # This may complain a bit more about missing libraries, which again should be copied into rt using rtcp. Eventually, it should print "9".
  13. sudo scripts/trace-execs
  14. find rt -perm -o+w # This should list exactly four files: rt/t.o, rt/t.s, rt/t.cpp, and rt/t.
  15. find rt -user nobody # This should find no files at all.
  16. sudo ./Bot "geordi-clone << 'x'" # If this does not print 'x', something is messed up. (Replace "geordi-clone" with the configured nick, of course.)
  17. If you want, you can now rm -f rt/usr/bin/strace rt/usr/bin/g++
  18. To get an idea of what rt could look like, look at the tree and file listing below.
  19. The bot can now be started with sudo ./Bot
  20. Geordi does not auto-reconnect. For that, just use something like while true; do ./Bot; sleep 120; done
